PageExtractor¶

class batukh.torch.segmenter.PageExtractor(use_pretrained=True, lock_pretrained=True)[source]¶

This class is used to detect boundary of text in an image of a document.

Parameters

  • use_pretrained (bool, optional) – whether to use the parameters of pre-trained resnet50 model. Default: True.

  • lock_pretrained (bool, optional) – whether to lock the layers of pretrained model. Only needed if use_pretrained is True. Default: True.

load_data(train_path, val_path=None, transform='default')[source]¶

Loads the data and creates a dataloader.

The label images should be:

  • the same name as the their original images (including the extension).

  • the same size as their original images.

  • black background with the page area filled with red color.

Parameters

  • train_path (str) – path to a directory containing two folders named originals and labels for training data.

  • val_path (str, optional) – path to a directory containing two named originals and labels for validation data. Default: None.

  • transform (transforms, optional) – Transforms to be applied on images and labels. It should be able to take a list of images as inputs and return a list of transformed images.

train(n_epochs, train_dl=None, val_dl=None, batch_size=1, shuffle=True, num_workers=4, pin_memory=True, criterion=None, optimizer=None, weight_decay=0, learning_rate=0.0001, learning_rate_decay=None, save_checkpoints=True, checkpoint_freq=None, checkpoint_path=None, max_to_keep=5, log_dir='./tensorboard_logs', log_freq=1000, device=None)[source]¶

Training method to detect boundary.

train_dl and val_dl should be provided if custom dataloaders are required. Although, most of the times, load_data() will do the job.

Note

If train_dl and/or val_dl are provided, then any already made dataloader using load_data() will not be used.

Parameters

  • n_epochs (int) – number of epochs.

  • train_dl (DataLoader, optional) – data loader to train on. Default: None.

  • val_dl (DataLoader, optional) – data loader to validate on. Default: None.

  • batch_size (int, optional) – batch size of the data loader created by load_data(). Default: 1.

  • shuffle (bool, optional) – whether to shuffle the data loader created by load_data().

  • num_workers (int, optional) – number of processes to load data. Default: 4

  • pin_memory (bool, optional) – whether to pin memory while loading data. Default: True.

  • criterion (_Loss or _WeightedLoss, optional) – The loss function to use. Default: CrossEntropyLoss(reduction="mean")

  • optimizer (Optimizer, optional) – The optimizer to be used to update the parameters. Default: Adam(model.parameters(), lr=learning_rate)

  • weight_decay (float, optional) – L2 regularization parameter for the optimizer(if it applies). Default: 0.

  • learning_rate (float, optional) – Learning rate to be used for the default optimizer. Default: 0.0001.

  • learning_rate_decay (float, optional) – exponential decay to be used on learning rate. Default: None.

  • save_checkpoints (bool, optional) – Whether to save training checkpoints. Default: True.

  • checkpoint_freq (int, optional) – The saving frequency. After each these many epochs, a checkpoint will be saved. Default: \(\left \lfloor\frac{\text{n_epochs}}{10}\right \rfloor + 1\).

  • checkpoint_path (str, optional) – Path of the checkpoint folder. Default: “./”

  • max_to_keep (int or None, optional) – The maximum number of latest checkpoints to be saved. None to save all. Default: 5.

  • device (str, optional) – The device to do computations on. Default: GPU, if GPU is available, else CPU.
